# 🎮 Unity 2D Learning Project

This repository contains my **first Unity game development project**, created as a **first-year college student** learning the fundamentals of game development.

> ⚠️ This is a **learning project**, not a production-ready or commercial game.

---

## 📌 Purpose

The goal of this project is to:

* Learn how to use the Unity Engine from scratch
* Understand how games are structured internally
* Practice programming with C# in a game context
* Learn version control using **Git & GitHub**
* Build a strong foundation for future game development

---

## 🧠 What This Project Covers

### 🧩 Unity Fundamentals

* Scene, Hierarchy, Inspector, and Game View
* GameObjects and Component-based architecture
* Navigating and using the Unity Editor

---

### 🏃 Player Movement System

* Horizontal movement with smooth acceleration
* Rigidbody2D-based physics movement
* Direction-based sprite flipping

---

### 🪂 Jump System

* Variable jump height (hold-to-jump)
* Double jump support
* Jump timing using `Time.time`

---

### 🌍 Physics & Collision

* Rigidbody2D for physics simulation
* Collider2D for collision detection
* Ground detection using raycasts

---

### 🎨 Sprite Handling

* Importing and using 2D sprites
* Setting pixels-per-unit correctly
* Tiling sprites to create environment

---

### 🎮 Animation Integration

* Animator usage for:

  * Grounded state
  * Movement speed
* Dynamic sprite flipping

---

### 🔊 Audio Feedback

* Jump sound effects
* Pitch variation for multiple jumps

---

### 🛠️ Development Workflow

* Unity project setup and version selection
* Asset importing and organization
* Using **Git & GitHub** for version control

---

## 🔁 Version Control

This project uses **Git & GitHub** instead of Unity’s built-in source control tools.

* Tracks all changes to scripts and project setup
* Allows safe experimentation and rollback
* Makes the project portable across systems

> This is part of learning real-world development workflows early.

---

## 🚫 What This Project Does NOT Cover

This project intentionally does **not** include:

* Creating custom art (sprites, animations)
* Advanced game architecture or design patterns
* UI systems (menus, HUD, etc.)
* Enemy AI or complex gameplay systems
* Optimization or production pipelines

---

## ⚙️ Technologies Used

* **Unity (2D setup / URP or Core)**
* **C# (MonoBehaviour scripting)**
* **Git & GitHub (version control)**

4. Press ▶ Play
